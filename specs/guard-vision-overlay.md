# 잠입 시야 노출 — 위험 타일 틴트 (정찰 도구)

## 목적
잠입 시 적(가드/몬스터)의 시야를 플레이어가 인지할 수 있어야 한다. 단 갑자기 알게 되면
개연성이 없으므로, **인게임 "정찰 도구" 아이템을 가졌을 때만** 가드의 시야 범위
(=위험 타일)를 맵에 **붉게 틴트**한다. 핍진성 확보.

## 1. 정찰 도구 아이템
- 신규 아이템 `scout_lens`("올빼미 안경" 등). **잠입 퀘스트 수락 시 `GiveItem("scout_lens")` 로 지급** +
  의뢰 대사로 정당화("이 안경을 끼면 놈들의 시선이 보일 걸세").
- 단순화: **인벤토리에 보유하면 활성**(별도 장착 슬롯 불필요). 보유 여부로 오버레이 on/off.
- (infiltration_quest / vault_heist_quest 의 수락 전이 actions 에 GiveItem 추가.)

## 2. 위험 타일 계산
- 순수 함수 `danger_tiles(guards: &[(pos, facing, vision_radius)], map: &Map) -> HashSet<(usize,usize)>`:
  각 가드에 대해 맵 타일 중 `is_in_view(gpos, gfacing, tile, vision_radius, FOV_BACK, map)` 인
  타일을 모은 합집합. (Phase 1 방향 시야 그대로 재사용 — 등 뒤·벽 너머는 제외.)
- 가드/몬스터 = 적대 엔티티(Monster). 가드뿐 아니라 일반 몬스터 시야도 표시(잠입 일반화).

## 3. 표시 (위험 타일 틴트)
- 시스템 `update_guard_vision_overlay`: 플레이어가 `scout_lens` 보유 시,
  `danger_tiles` 의 각 타일 위에 **반투명 붉은 오버레이 엔티티**(타일 위 Z) 배치.
  미보유면 오버레이 전부 제거.
- 플레이어/가드 이동·facing 변경, 인벤토리 변경 시 갱신(매 턴 또는 변경 감지).
- 기존 타일 렌더는 건드리지 않고 별도 오버레이 엔티티 레이어로(깔끔). 색 예 `rgba(1,0,0,0.25)`.

## 4. 테스트 (100% 커버리지, 한글 의도서술형)
- `danger_tiles`: 가드 정면 타일 포함·등 뒤 제외, 벽 너머 제외, 여러 가드 합집합, 가드 없음 → 빈 집합.
- 오버레이 시스템: scout_lens 보유 시 위험 타일에 오버레이 스폰, 미보유 시 없음, 가드 facing/이동 시 갱신.
- 정찰 도구 지급: 잠입 퀘스트 수락 actions 에 GiveItem(scout_lens) 포함.

## 5. 결정 (기본값)
- 표시 = 위험 타일 붉은 틴트(오버레이 엔티티). 활성 = scout_lens 인벤토리 보유. 지급 = 잠입 퀘스트 수락 시.
- bevy-rogue 빌드를 점유하는 다른 작업(지형 변화 등) 종료 후 진행.
