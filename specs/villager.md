# 마을 주민 (Villager)

## 목적

마을 맵(마을 - 격자 배치, 마을 - 유기적 배치)에 자율 행동하는 주민 NPC를 추가하여 세계관 몰입감을 높인다.

## 동작 명세

### 스폰

- [x] 마을 맵에만 주민이 스폰된다 (던전·숲 맵에는 스폰되지 않는다)
- [x] 주민은 방(room) 중심 타일에 스폰된다
- [x] 맵이 재생성되면 기존 주민은 모두 제거되고 새로운 주민이 스폰된다
- [x] 마을이 아닌 맵으로 전환하면 주민이 모두 제거된다

### 턴제 이동

- [x] 이동은 실시간이 아닌 **턴제**로 동작한다
- [x] 플레이어가 **이동 애니메이션을 완료**하거나 주민과 부딪히면 플레이어 턴이 소비된다 — 이동 중(MovingTo 존재)에는 턴이 소비되지 않으며, smooth_player_lerp 가 목적지에 도달한 시점에 PlayerActedEvent 발행
- [x] 플레이어 턴이 소비되면 모든 주민이 각각 한 번 이동한다
- [x] 주민은 인접한 Floor 타일로 무작위 이동하거나 제자리에 머문다 (30% 확률로 제자리)
- [x] 주민은 Wall 타일로 이동하지 않는다
- [x] 플레이어와 충돌(대화)한 주민은 해당 턴에 이동하지 않는다

### 충돌

- [x] 플레이어는 주민이 점유한 타일로 이동할 수 없다
- [x] 주민은 다른 주민이 점유한 타일로 이동할 수 없다 (주민끼리 겹치지 않음)
- [x] 주민은 플레이어가 현재 점유한 타일로 이동할 수 없다 (overlap 방지)

### 대사

- [x] 플레이어가 주민 타일로 이동 시도(부딪힘)하면 대사를 로그에 출력한다
- [x] 대사는 주민마다 정해진 목록을 순서대로 순환한다
- [x] 같은 주민과의 매 충돌마다 다음 대사로 넘어간다

## 엣지 케이스

- 이동 가능한 인접 타일이 없으면 주민은 제자리에 머문다
- 대사 목록의 마지막 이후에는 처음 대사로 돌아간다
- 주민이 미탐색 타일로 이동해도 가시성 규칙에 따라 렌더링된다

## 퀘스트 NPC 스폰 규칙

- [x] 퀘스트 NPC는 퀘스트별로 최대 1명만 스폰된다 (동일 quest_id 중복 없음)
- [x] 퀘스트 NPC를 먼저 배치한 후, 남은 방은 일반 NPC로 순환 채운다
- [x] 퀘스트 NPC는 스폰된 방(`home_room: Some(Rect)`) 밖으로 이동하지 않는다; 일반 NPC는 제한 없음

## 퀘스트 NPC 글리프

퀘스트 상태에 따라 4가지로 표시:

| 글리프 | 색상 | 조건 |
|--------|------|------|
| `?` | 노란색 `rgb(1.0, 0.9, 0.1)` | 퀘스트 있음 — initial_phase 이거나 아직 수락 전 |
| `?` | 초록색 `rgb(0.3, 1.0, 0.6)` | 퀘스트 수락, 다음 페이즈로 넘어갈 수 없음 — 아이템 수집·이동 중 |
| `!` | 초록색 `rgb(0.3, 1.0, 0.6)` | 다음 페이즈로 넘어갈 수 있음 — `on_interact` 에 `AdvancePhase` 가 포함(Branch 내부 포함)되거나 `auto_advance` 조건 충족. Log·Branch(Log) 등 순수 힌트 액션만 있는 경우는 `?` 로 표시 |
| `v` | 기본 색상 | 터미널 (퀘스트 완료 또는 퀘스트 없는 NPC) |

- [x] 퀘스트가 없는 주민은 항상 `v`로 표시한다
- [x] 퀘스트 NPC는 스폰 시 노란 `?`로 초기화된다 (퀘스트 수락 전)
- [x] `QuestState`가 변경될 때마다 글리프를 재계산한다 (`update_villager_glyph` 시스템) — `handle_bump` 이후 실행되도록 순서를 보장해야 같은 프레임 내 변경 틱 충돌로 인한 갱신 누락을 방지할 수 있다
- [x] 맵 진입 시 새로 스폰된 주민(`Added<Villager>`)도 즉시 글리프를 현재 `QuestState` 기준으로 갱신한다 (노란 `?` 고정 방지)
- [x] `Villager.base_color`에 원래 색상을 저장하여 터미널 복귀 시 사용한다

## 등장 주민 유형

| 초기 글리프 | 색상 | 이름 | 대사 수 | 주제 영역 |
|-------------|------|------|---------|-----------|
| `?` (노란) | 노란 | 장로 | (퀘스트 전용) | gem_quest |
| `?` (노란) | 청록 | 연금술사 | (퀘스트 전용) | alchemist_quest |
| `v` | 금색 | 촌장 | 32줄 | 마을 행정·방문자·공동체 규칙 |
| `v` | 연두 | 상인 | 32줄 | 거래·교역·희귀 물자·먼 지역 소식 |
| `v` | 갈색 | 농부 | 32줄 | 농사·날씨·계절·가축·자연 |
| `v` | 흰색 | 아이 | 32줄 | 놀이·호기심·가족·모험 동경 |
| `?` (노란) | 회청 | 노인 | 32줄 | 역사·봉인 암시·세계 위기·삶의 지혜 (world_fracture) |

### 대사 설계 원칙

- [x] NPC 간 대사 주제가 겹치지 않도록 캐릭터별 전담 영역 구분
- [x] 각 일반 NPC(퀘스트 없는 NPC 포함 노인)는 32줄 이상의 대사 보유
- [x] 퀘스트 NPC(장로·연금술사)는 일반 대사 없이 퀘스트 다이얼로그만 사용

## 구현 범위

- `MapType` 열거형 (`Dungeon`, `Village`) — `Map` 구조체에 포함
- `OccupiedTiles` 리소스 — PreUpdate에서 주민 위치 동기화, 플레이어 이동 차단에 사용
- `PlayerActedEvent` — 플레이어 행동 시 발행, 주민 턴 트리거
- `BumpTileEvent(x, y)` — 플레이어가 주민 타일 충돌 시 발행, 대사 트리거
- `PlayerSystemSet::Movement` — 주민 시스템이 플레이어 이동 이후 실행되도록 순서 보장
- `src/modules/villager/mod.rs` — 주민 스폰·턴 이동·충돌·대사 시스템
