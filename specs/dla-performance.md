# DLA 생성기 성능 최적화

## 목적

DLA(Diffusion-Limited Aggregation) 맵 생성 시 발생하는 수 초의 지연을 제거한다.

## 문제 원인

- `random_floor_tile()` 이 매 파티클마다 전체 타일(16,000개)를 선형 스캔
- 목표 타일 수 5,600개 × O(16,000) ≈ **9천만 번** 반복

## 동작 명세

- [x] DLA 생성이 체감 지연 없이 완료된다
- [x] 생성된 맵의 Floor 비율은 35% 이상을 유지한다
- [x] 맵 경계는 모두 Wall로 유지된다

## 해결 방법

- Floor 타일 목록을 `Vec<(usize, usize)>`로 별도 유지 → O(1) 랜덤 선택
- 새 타일 추가 시 `push()`로 목록 갱신
- 파티클 최대 스텝을 800 → 400으로 축소 (반경 ≈ √400 = 20으로 충분)
- 미사용 `random_floor_tile()` 함수 제거
