# 몬스터 시야 (Monster FOV)

## 목적

몬스터가 플레이어를 무조건 추적하는 대신, 각자의 시야 반경과 시선(LOS)을 가지고
플레이어가 시야에 들어올 때만 추적하도록 하여 전술적 긴장감을 높인다.

## 동작 명세

### 시야

- [x] 각 몬스터는 고유한 `vision_radius`를 갖는다
- [x] 시야 판정: 플레이어까지의 거리 ≤ `vision_radius` **AND** Bresenham LOS가 통과될 때 탐지
- [x] LOS는 `map::is_line_of_sight_clear`를 공유 사용한다

### AI 상태

- [x] `Idle` — 플레이어를 인지하지 못한 상태. 무작위 배회 (30% 확률 제자리)
- [x] `Alerted` — 플레이어를 발견한 상태. 플레이어 방향으로 이동 또는 인접 시 공격
- [x] 시야에 플레이어가 들어오면 `Idle → Alerted`, `alert_turns = MAX_ALERT` 리셋
- [x] `Alerted` 상태에서 플레이어가 시야를 벗어나도 `alert_turns`가 남아 있으면 계속 추적
- [x] `alert_turns`가 0이 되면 `Alerted → Idle`

### 몬스터별 시야 반경

| 몬스터 | vision_radius | 특성 |
|--------|--------------|------|
| 고블린 | 6 | 좁은 시야 |
| 오크   | 8 | 넓은 시야 |
| 트롤   | 5 | 가장 좁은 시야 |

### 상수

- `MAX_ALERT_TURNS = 5` — 시야를 잃은 후 추적을 유지하는 최대 턴 수

## 엣지 케이스

- 몬스터가 벽 뒤에 있을 때는 거리 내라도 탐지하지 않는다 (LOS 차단)
- 이미 Alerted 상태에서 플레이어가 다시 시야에 들어오면 `alert_turns` 리셋
- Idle 상태에서도 인접하면 공격은 수행한다 (우발적 접촉)
