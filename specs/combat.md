# 적 & 전투 (Enemy & Combat)

## 목적

던전 맵에 자율 행동하는 적 몬스터를 배치하고, 플레이어-몬스터 간 턴제 근접 전투를 구현한다.
"인접 타일로 이동 시도 = 공격" 규칙으로 이동과 전투를 통합한다.

## 동작 명세

### 스폰

- [ ] 던전 맵에만 몬스터가 스폰된다 (마을·숲 맵에는 스폰되지 않는다)
- [ ] 몬스터는 방(room) 중심 타일에 스폰된다 (플레이어 스폰 방 rooms[0] 제외)
- [ ] 맵이 재생성되면 기존 몬스터는 모두 제거되고 새로운 몬스터가 스폰된다

### 전투 스탯

- [ ] 플레이어 초기 스탯: HP 30, 공격력 5, 방어력 1
- [ ] 데미지 = max(1, 공격자.공격력 − 방어자.방어력)
- [ ] 몬스터는 유형별로 스탯이 다르다 (아래 표 참조)

### 공격

- [ ] 플레이어가 몬스터 타일로 이동을 시도하면 근접 공격이 발동된다 (이동하지 않음)
- [ ] 몬스터가 플레이어에 인접(4방향)한 경우 플레이어를 공격한다 (이동하지 않음)
- [ ] 공격 결과(데미지, 남은 HP)가 로그에 출력된다
- [ ] 플레이어가 몬스터를 공격해도 턴이 소비된다 (몬스터들이 반응 행동)

### 몬스터 AI (턴제)

- [ ] 플레이어 턴 종료 후 살아있는 모든 몬스터가 각각 한 번 행동한다
- [ ] 플레이어와 인접한 경우 공격한다
- [ ] 인접하지 않은 경우 플레이어 방향으로 한 칸 이동한다 (Manhattan 거리 최소화)
- [ ] 몬스터는 다른 몬스터가 점유한 타일로 이동하지 않는다
- [ ] 이동 가능한 타일이 없으면 제자리를 유지한다

### 사망

- [ ] HP가 0 이하가 되면 해당 엔티티는 다음 정리 단계에서 제거된다
- [ ] 몬스터 사망 시 "[몬스터명]을(를) 처치했다!" 로그 출력
- [ ] 플레이어 사망 시 "당신은 죽었습니다." 로그 출력 후 이동 불가 상태로 전환
- [ ] 같은 턴에 죽은 몬스터는 그 턴 행동을 건너뛴다

## 등장 몬스터 유형

| 글리프 | 색상 | 이름   | HP | 공격 | 방어 |
|--------|------|--------|----|------|------|
| `g`    | 연두 | 고블린 |  6 |    3 |    0 |
| `O`    | 주황 | 오크   | 10 |    5 |    2 |
| `T`    | 청록 | 트롤   | 16 |    8 |    3 |

## 전투 예시

```
플레이어(ATK:5, DEF:1) vs 고블린(HP:6, ATK:3, DEF:0)

플레이어 공격: 데미지 = max(1, 5 - 0) = 5
  → "고블린에게 5 데미지! (HP: 1/6)"
플레이어 재공격: 데미지 = 5
  → "고블린을(를) 처치했다! (5 데미지)"

고블린 공격: 데미지 = max(1, 3 - 1) = 2
  → "고블린에게 2 데미지! (HP: 28/30)"
```

## 구현 범위

- `CombatStats { hp, max_hp, attack, defense }` — 플레이어 및 몬스터 공통 컴포넌트 (`combat` 모듈)
- `Defeated` — 플레이어 사망 마커 컴포넌트 (`combat` 모듈), 이동 차단에 사용
- `calc_damage(attack, defense) -> i32` — 데미지 계산 순수 함수 (`combat` 모듈)
- `MonsterTiles` 리소스 — PreUpdate에서 몬스터 위치 동기화, 플레이어 이동 차단 (`map` 모듈)
- `AttackMonsterEvent(x, y)` — 플레이어가 몬스터 타일 충돌 시 발행 (`map` 모듈)
- `MonsterRespawnEvent { map_type, rooms }` — 맵 재생성 시 몬스터 재스폰 트리거 (`map` 모듈)
- `src/modules/combat/mod.rs` — 공유 전투 타입·순수 함수
- `src/modules/monster/mod.rs` — 몬스터 스폰·AI·전투 시스템
