# 추가 시스템: 액티브 스킬 · 함정 · 조명/그림자 · 퀘스트 저널

사용자 추천 진행분. 각각 독립 기능이며 spec→test→구현, nightly 100% 커버리지.
순서: (지형/폭발 코어 완료 후) A → B → C → D. (D 는 의존 적어 먼저도 가능.)

## A. 액티브 스킬/마법 (MP 활용)
`CombatStats.mp/max_mp` 가 이미 있으나 거의 안 쓰임 → 액티브 스킬로 활용.
- 스킬 정의(데이터): id, 한글명, `mp_cost`, 효과. 코드 enum 또는 RON.
- 초기 3종:
  - **파이어볼**: 조준 타일에 **`ExplosionEvent { terrain:true, entity_damage }`** 발행(지형 파괴 + 범위 피해) + 화염 원소 부여. (지형 폭발 코어 재사용 → 의존.)
  - **점멸(Blink)**: 시야 내 빈(통과 가능) 타일로 순간이동.
  - **치유**: MP 소모 → HP 회복.
- 입력: 단축키(예 1/2/3) 또는 스킬창. MP 부족 시 불가, 사용 시 MP 차감 + 턴 소비(PlayerActedEvent).
- 순수 함수로 효과/비용 판정 분리(`can_cast(mp,cost)`, 효과 계산). 테스트: MP 소모/부족, 각 스킬 효과, 파이어볼→ExplosionEvent, 점멸 목적지 유효성, 치유 max 클램프.

## B. 함정 (Trap)
- `TrapKind`: **Spike**(피해), **Poison**(독 상태부여), **Alarm**(`stealth_blown` + 주변 가드 경계), **Teleport**(랜덤 통과타일로 이동).
- `Trap { kind, hidden: bool }` 컴포넌트(타일 위 엔티티). 밟으면(플레이어/몬스터 진입) 발동. 숨김 함정은 인접/탐지 시 노출. (선택) 해제: 확률 or 도구.
- 배치: 생성기/퀘스트(특히 잠입 구역에 Alarm). 잠입 시너지: 경보 함정 = 탐지 트리거.
- 테스트: 발동 효과별, 숨김/노출, alarm→stealth_blown+경계, 텔레포트 목적지 통과가능.

## B-2. 플레이어 함정 설치/해제 (B 기본 시스템 후속)
- **설치**: "함정 키트" 아이템(소비 또는 회수형)으로 인접/현재 타일에 플레이어 함정 배치. 이 함정은 **몬스터 진입 시 발동**(피해/기절/독 등)으로 전술·잠입에 활용. 플레이어 자신은 안 밟게(아군 함정 표시).
- **해제**: 탐지한(노출된) 함정을 해제 액션(키)으로 제거 — 해제 도구 보유 시 확정, 없으면 확률. 성공 시 (회수형이면) 키트 회수.
- B 기본 trap 컴포넌트/발동 로직 재사용. 순수 함수로 설치 가능/해제 판정 분리.
- 테스트: 설치(빈 타일/점유 거부), 몬스터가 플레이어 함정 발동, 해제(도구 유/무 확률), 키트 소모/회수.

## C. 조명/그림자
- 타일 광량(밝음/어둠) — 광원(횃불/플레이어 시야광) 반경 기반(단순: 광원 반경 내 밝음).
- 잠입 연계: **플레이어가 어둠에 있으면 가드 탐지 반경 감소**(can_see_player 에 광량 반영), 밝은 곳은 노출↑. 어둠은 은신 보너스.
- 렌더: 어두운 타일 디밍(기존 visibility 디밍과 통합).
- 순수 함수 `light_level(tile, sources)` / 탐지 보정. 테스트: 광량 계산, 어둠 속 탐지 감소, 광원 반경 경계.
- (FOV/렌더와 얽힘 — 방향 시야·지형 이후.)

## F. 인게임 피처 도감/가이드 패널
조작키만 있는 Help 를 "조작 + 피처 도감"으로 확장(또는 별도 토글 패널).
- **데이터 주도 가이드 항목**: `(카테고리, 제목, 설명[, 간단 텍스트 시각화])` 목록. 카테고리 예: 방향 시야/잠입, 원소 반응, 아이템 티어·레어도, 액티브 스킬, 함정, 파밍·드롭, 전투.
- 토글 키(예 Help 확장 또는 `F1`/`?`와 충돌 없는 키)로 패널 열고, 카테고리/항목 탐색(↑↓). 읽기 전용.
- 기존 Help(`ui/help.rs`)·퀘스트 저널(D) 패널 UI 틀 재사용.
- 순수 함수 `guide_entries() -> Vec<GuideEntry>` / 항목 렌더 텍스트 빌더로 테스트. (풍부한 인터랙티브 시각화는 docs/visuals 브라우저용 유지 — 인게임은 텍스트 중심.)
- 테스트: 항목 목록·카테고리, 토글, 선택 항목 렌더.

## E. 고정 상점 NPC (가판대)
현재 상인이 일반 빌리저처럼 돌아다녀 "물건 파는 사람"임이 안 드러남 → 상인을 가판대 뒤에 고정.
- **정지(stationary)**: 상인(vendor) 빌리저는 이동 안 함. villager def 에 `stationary`/`vendor` 속성 추가 → `villager_turn` 이 vendor 는 스킵(제자리 유지).
- **가판대(counter)**: 마을 생성 시 한 건물을 "상점"으로 — 통행 불가 **카운터**(신규 타일 또는 가구 엔티티) 한 줄 + 그 **뒤에 vendor 고정 배치**. 플레이어는 카운터 앞에 서서 거래.
- **상호작용**: 카운터(또는 그 너머 vendor) 에 인접해 interact → 해당 vendor 의 상점 열림(기존 `ui/shop.rs`). 카운터가 막아도 **카운터 앞 타일에서 열리게** 보정.
- **시각 구분**: vendor 글리프 구분(예 `$`/상인 아이콘 + 고유 색), 카운터 타일/엔티티 구분, (선택) 간판. 미니맵에 상점 마커.
- 테스트: vendor 미이동, 카운터 뒤 배치, 카운터 앞 상호작용으로 상점 열림, 시각/마커 구분.

## D. 퀘스트 저널 UI
- 활성 퀘스트 목록 + 각 현재 페이즈 `objective` 를 패널로(키 토글, 예 `J`). 기존 `QuestState.phases` + `QuestDef.phases[].objective` + 미니맵 마커 활용. 읽기 전용.
- 순수 함수 `journal_entries(quest_state, registry) -> Vec<(title, objective)>`(활성/진행 중만). 테스트: 활성 퀘스트만 표시, 페이즈별 objective, 완료/미시작 제외, 빈 경우.
- 의존 적음 — 독립 진행 가능.
