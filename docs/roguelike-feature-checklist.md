# 로그라이크 기본 기능 검토

이 문서는 현재 `bevy-rogue`가 갖춘 로그라이크 기본 요소와, 플레이 경험을 완성하기 위해 보완하면 좋은 기능을 정리한다. 개별 기능의 상세 구현 명세는 필요할 때 `specs/` 아래에 별도 문서로 분리한다.

## 현재 갖춘 기반

| 영역 | 현재 상태 |
|------|-----------|
| 턴 진행 | 플레이어 행동 완료 후 `PlayerActedEvent`를 기준으로 몬스터, 주민, 저장, 지속성 처리가 이어진다. |
| 이동 | 방향키/WASD 8방향 이동, 제자리 대기, 연속 이동, 마우스 경로 이동을 지원한다. |
| 시야 | 방향(facing) 기반 두-반원 FOV(앞 8 / 뒤 3), 몬스터도 동일 모델로 탐지, 발견/가시 타일 분리. |
| 전투 | 근접 충돌 기반 공격, 공격/방어 스탯, 몬스터 AI, 활 원거리 조준, 원소 반응, 사망 마커, 피격 피드백. |
| 아이템 | 드롭, 자동 픽업, 장비, 소모품, 퀘스트 아이템, 무기/방어구 랜덤 스탯·레어도 파밍, 글리프 스타일 전환. |
| UI | 상단 HUD, 하단 로그, HP/MP 바, 장비 패널, 퀘스트 패널, 상점, 미니맵, Game Over 오버레이, 인게임 도움말, 피처 도감/가이드 패널. |
| 월드 구조 | 마을/숲/던전/Named 존 전환, 포탈, 존별 맵 캐시와 지속성, 23종 절차 생성기(Water/Sand 수상 맵 포함). |
| 퀘스트 | RON 기반 정의, 자동 진행, 분기, 플래그, 퀘스트 스폰, Named 존 연동, 잠입(스텔스) 퀘스트(가드 탐지·무탐지 보너스). |
| 저장 | 매 턴 자동 저장/자동 로드, 탐험 타일 비트팩, 글로벌 시드 기반 맵 복원이 있다. |

## 우선 보완 항목

### 1. 게임 오버와 새 게임 흐름

상세 스펙: [`specs/game-over-new-run.md`](../specs/game-over-new-run.md)

현재 상태: 구현됨.

구현된 UX:

- 사망 시 중앙 Game Over 오버레이 표시
- 현재 존과 생존 턴 표시
- `R` 또는 `N`으로 기존 세이브를 삭제하고 새 게임 시작
- `Esc`로 게임 종료
- 새 게임 시작 시 플레이어, 월드, 퀘스트, 인벤토리, 장비, 존 지속성, 로그, 패널 상태 초기화
- 자동 로드 때문에 죽은 상태가 계속 복원되는 문제 방지

남은 개선 후보:

- 처치 수, 획득 골드, 완료 퀘스트 등 run summary 추가
- `R`은 마지막 체크포인트 재시도, `N`은 완전 새 게임처럼 의미를 분리

우선도: 완료. 로그라이크 핵심 루프의 최소 UX가 갖춰졌다.

### 2. 인게임 도움말 / 키 바인딩 오버레이

상세 스펙: [`specs/help-overlay.md`](../specs/help-overlay.md)

현재 상태: 구현됨.

구현된 UX:

- `?` 또는 `H`로 도움말 패널 열기/닫기
- 이동, 대기, 전투, 장비, 퀘스트, 상점, 미니맵, 기타 키를 섹션별 표시
- 패널이 열린 동안 키보드 이동과 마우스 경로 이동 차단
- `Esc`로 닫기

남은 개선 후보:

- 실제 키 바인딩을 단일 데이터 소스로 관리하여 문서와 오버레이 중복 제거
- 도움말 패널 내 스크롤 또는 카테고리 탭 추가

우선도: 완료. 신규 플레이어가 외부 문서 없이 기본 조작을 확인할 수 있다.

### 2-1. 피처 도감 / 가이드 패널

상세 스펙: [`specs/additional-systems.md`](../specs/additional-systems.md) §F

현재 상태: 구현됨.

구현된 UX:

- `F2`로 도감 패널 열기/닫기 (도움말 `H`/`?`·맵생성 `F1`·퀘스트 `J`/`Q`와 충돌 없음)
- 게임의 실제 기능을 카테고리·항목으로 정리한 **읽기 전용** 도감
  (방향 시야/잠입, 위험 타일·정찰 도구, 조명/그림자 은신, 원소 반응, 아이템 티어·레어도,
  파밍·드롭, 액티브 스킬, 함정 종류·설치/해제, 파괴 지형·폭발, 고정 상점, 전투, 퀘스트/저널)
- `↑`/`↓`로 항목 선택(경계에서 순환), 선택 항목의 설명·간단 텍스트 시각화가 펼쳐짐
- 패널이 열린 동안 이동·스킬·함정 입력 차단(다른 모달 패널과 동일한 흐름)
- 데이터는 순수 함수 `guide_entries()`/`guide_categories()`, 렌더는 `build_guide_sections()`로 분리
- 설명은 코드/스펙에 구현된 실제 동작(키·수치·메커니즘)을 근거로 작성

남은 개선 후보:

- 도움말(조작)과 도감(메커니즘)을 탭 하나로 통합
- 항목별 인터랙티브 시각화(docs/visuals 수준)는 인게임 텍스트에서 제외 유지

우선도: 완료. 신규 플레이어가 외부 문서 없이 핵심 메커니즘을 인게임에서 확인할 수 있다.

### 3. 상단 HUD / 상태 요약

상세 스펙: [`specs/status-hud.md`](../specs/status-hud.md)

현재 상태: 구현됨.

구현된 UX:

- 현재 존 이름
- 현재 턴
- HP/MP 숫자
- 공격력/방어력
- 골드
- 장착 무기/방어구 요약
- 현재 맵 생성기

남은 개선 후보:

- 몬스터 처치 수, 퀘스트 목표, 상태이상 등 run context 추가
- 패널이 열릴 때 HUD와 겹치지 않도록 레이아웃 세부 조정

우선도: 완료. 플레이어가 매 턴 판단할 핵심 정보를 한눈에 볼 수 있다.

### 4. 목표 안내와 퀘스트 추적

상세 스펙: [`specs/quest-guidance.md`](../specs/quest-guidance.md)

현재 상태: 일부 구현됨.

구현된 UX:

- 퀘스트 패널에서 현재 phase의 목표 존 표시
- 목표 존이 현재 위치와 같으면 `현재 위치` 힌트 표시
- 현재 phase에 스폰되는 퀘스트 아이템의 보유/필요 수량 표시
- phase가 NPC 상호작용으로 진행될 때 `다음: NPC와 대화` 힌트 표시
- `InZone` 자동 진행 조건을 위치 힌트로 추출

남은 개선 후보:

- 퀘스트 목표 아이템 미니맵 마커 연동
- 발견된 퀘스트 제공자 NPC의 현재 존/미니맵 표시 안내
- 완료 가능한 NPC의 방향 또는 미발견 NPC 위치 추론
- 여러 목표가 있는 phase에서 우선순위/완료 상태를 더 세밀하게 표시
- 포탈이 아직 열리지 않은 Named 존의 입구 위치 안내

우선도: 진행 중. 퀘스트 패널이 외부 문서 없이도 다음 행동을 더 직접적으로 안내하기 시작했다.

## 로그라이크 기본기로 추가하면 좋은 항목

### 성장 루프

상세 스펙: [`specs/player-xp-progression.md`](../specs/player-xp-progression.md)

현재 상태: 일부 구현됨.

구현된 UX:

- 몬스터 처치 시 몬스터 종류별 XP 획득
- XP 획득과 레벨업 메시지를 로그에 표시
- HUD에 현재 레벨과 XP/다음 레벨 필요량 표시
- 레벨업 시 최대 HP/MP 증가 및 HP/MP 완전 회복
- 레벨, XP, 처치 수를 자동 저장/로드와 새 게임 초기화 흐름에 포함

남은 개선 후보:

- 공격/방어/속도 중 하나를 선택하는 레벨업 보상 UI
- 퀘스트 완료 보상으로 XP 지급
- 몬스터 처치 수와 성장 상태를 Game Over run summary에 표시
- 장비 보정과 분리된 기본 공격/방어 스탯 모델 추가

우선도: 진행 중. 반복 플레이 동기를 만드는 최소 성장 루프가 들어갔다.

### 원거리 전투와 타겟팅

현재 활은 장비 종류로 존재하지만 전투는 근접 이동 충돌 중심이다. 원거리/마법이 들어가면 전투 선택지가 확장된다.

후보 설계:

- `f`로 타겟팅 모드 진입
- 가장 가까운 보이는 몬스터 자동 선택
- 방향키/Tab으로 대상 변경
- Enter로 발사, Esc로 취소
- 활 장착 시 사거리와 피해 보정 적용
- MP를 사용하는 주문으로 확장 가능

우선도: 중간. 전투 깊이를 크게 늘리지만 UI/AI/밸런스와 같이 설계해야 한다.

### 아이템 조작 확장

현재 아이템은 자동 픽업과 장비/소모품 사용 중심이다. 로그라이크 기본 상호작용을 늘리려면 아이템 관리 기능이 더 필요하다.

후보 기능:

- 바닥 아이템 목록
- 선택 픽업
- 아이템 버리기/drop
- 소모품 퀵슬롯
- 수량 선택
- 장비 비교 표시

우선도: 중간. 인벤토리 복잡도가 커지기 전 UX 틀을 잡는 것이 좋다.

### 휴식과 회복

`Space` 대기는 있지만, 여러 턴 휴식하거나 안전할 때까지 회복하는 흐름은 없다.

후보 기능:

- `.` 또는 `Space`: 한 턴 대기 유지
- `R`: 안전할 때까지 휴식
- 적이 시야에 들어오거나 HP가 가득 차면 자동 중단
- 휴식 중 HP/MP 자연 회복 규칙

우선도: 중간. 던전 탐험 템포와 자원 관리가 생긴다.

## 깊이를 더하는 후순위 항목

### 지형과 상호작용

현재 타일 의미는 주로 `Wall`/`Floor`에 머문다. 맵 생성기는 다양한데, 타일 상호작용은 아직 단순하다.

후보:

- 문, 잠긴 문, 열쇠
- 함정
- 물, 독 늪, 불, 얼음
- 시야를 가리는 안개/연기
- 파괴 가능한 벽 또는 장애물

### 상태이상

전투 결과가 HP 감소 중심이라 장기적인 전술 변수가 적다.

후보:

- 독, 출혈, 화상
- 둔화, 가속
- 기절, 침묵
- 방어력 감소, 취약
- 축복/저주 아이템

### 로그/메시지 히스토리

하단 로그는 최근 몇 줄만 보여준다. 전투와 퀘스트 메시지가 늘어나면 과거 메시지를 다시 확인할 수 있어야 한다.

후보:

- 로그 히스토리 패널
- 중요도별 색상
- 반복 메시지 압축
- 퀘스트/전투/시스템 필터

### 설정과 접근성

이미 글리프 스타일 전환이 있어 좋은 출발점이다. 장시간 플레이를 위해 설정 범위를 넓힐 수 있다.

후보:

- 폰트 크기 조절
- 애니메이션 속도 조절
- 색각 보정 팔레트
- 키 리맵
- 로그 표시 줄 수 조절

## 추천 구현 순서

1. Game Over 오버레이와 새 게임 흐름 (완료)
2. 인게임 도움말 패널 (완료)
3. 상단 HUD / 상태 요약 (완료)
4. 퀘스트 목표 안내 개선 (일부 구현)
5. 레벨업 성장 루프 (일부 구현)
6. 아이템 조작 확장
7. 휴식/회복 규칙
8. 지형 상호작용과 상태이상

이 순서는 새 콘텐츠를 많이 추가하기보다, 이미 구현된 시스템을 플레이어가 더 잘 이해하고 반복 플레이할 수 있게 만드는 방향을 우선한다.
