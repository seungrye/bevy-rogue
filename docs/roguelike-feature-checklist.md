# 로그라이크 기본 기능 검토

이 문서는 현재 `bevy-rogue`가 갖춘 로그라이크 기본 요소와, 플레이 경험을 완성하기 위해 보완하면 좋은 기능을 정리한다. 개별 기능의 상세 구현 명세는 필요할 때 `specs/` 아래에 별도 문서로 분리한다.

## 현재 갖춘 기반

| 영역 | 현재 상태 |
|------|-----------|
| 턴 진행 | 플레이어 행동 완료 후 `PlayerActedEvent`를 기준으로 몬스터, 주민, 저장, 지속성 처리가 이어진다. |
| 이동 | 방향키/WASD 8방향 이동, 제자리 대기, 연속 이동, 마우스 경로 이동을 지원한다. |
| 시야 | 플레이어 FOV와 몬스터 FOV가 있으며, 발견/가시 타일이 분리되어 있다. |
| 전투 | 근접 충돌 기반 공격, 공격/방어 스탯, 몬스터 AI, 사망 마커, 피격 피드백이 있다. |
| 아이템 | 드롭, 자동 픽업, 장비, 소모품, 퀘스트 아이템, 글리프 스타일 전환이 있다. |
| UI | 하단 로그, HP/MP 바, 장비 패널, 퀘스트 패널, 상점, 미니맵이 있다. |
| 월드 구조 | 마을/숲/던전/Named 존 전환, 포탈, 존별 맵 캐시와 지속성이 있다. |
| 퀘스트 | RON 기반 퀘스트 정의, 자동 진행, 분기, 플래그, 퀘스트 스폰, Named 존 연동이 있다. |
| 저장 | 매 턴 자동 저장/자동 로드, 탐험 타일 비트팩, 글로벌 시드 기반 맵 복원이 있다. |

## 우선 보완 항목

### 1. 게임 오버와 새 게임 흐름

현재 플레이어가 죽으면 `Defeated` 상태로 이동이 막히고 로그에 사망 메시지가 남는다. 하지만 플레이어가 다음에 무엇을 할 수 있는지 화면에서 충분히 안내되지 않는다.

필요한 UX:

- 사망 시 중앙 Game Over 오버레이 표시
- 현재 턴, 현재 존, 처치 수 또는 진행 요약 표시
- `R` 재시작, `N` 새 게임, `Esc` 종료 같은 명확한 선택지
- 새 게임 시작 시 기존 세이브 삭제 또는 무시 흐름
- 자동 로드 때문에 죽은 상태가 계속 복원되는 문제 방지

우선도: 높음. 로그라이크에서 사망은 핵심 루프이므로 가장 먼저 정리할 가치가 크다.

### 2. 인게임 도움말 / 키 바인딩 오버레이

`docs/keybindings.md`는 있지만 게임 안에서 조작법을 확인하는 방법이 없다. 현재 조작이 많아졌기 때문에 신규 플레이어는 외부 문서를 보지 않으면 기능을 놓치기 쉽다.

필요한 UX:

- `?` 또는 `H`로 도움말 패널 열기/닫기
- 이동, 대기, 전투, 장비, 퀘스트, 상점, 미니맵, 개발용 키를 섹션별 표시
- 패널이 열린 동안 이동 입력 차단
- `Esc`로 닫기

우선도: 높음. 구현 비용 대비 플레이 가능성이 크게 좋아진다.

### 3. 상단 HUD / 상태 요약

HP/MP 바는 플레이어 위에 붙어 있지만, 전투 판단에 필요한 전체 상태가 항상 보이지는 않는다. 장비 패널을 열어야 공격/방어/인벤토리 상태를 자세히 볼 수 있다.

보완 후보:

- 현재 존 이름
- 현재 턴
- HP/MP 숫자
- 공격력/방어력
- 골드
- 장착 무기/방어구 요약
- 현재 맵 생성기 또는 던전 층

우선도: 높음. 플레이어가 매 턴 판단할 정보를 한눈에 볼 수 있어야 한다.

### 4. 목표 안내와 퀘스트 추적

퀘스트 패널에는 진행 중인 퀘스트와 objective가 표시된다. 다만 목표가 어느 존에 있는지, 목표 지점이 발견됐는지, 다음 행동이 NPC 대화인지 아이템 수집인지가 항상 명확하지는 않다.

보완 후보:

- 퀘스트 objective에 대상 존/조건 요약 추가
- 미니맵 마커와 퀘스트 목표 연동
- 완료 가능한 NPC의 방향 또는 현재 존 내 존재 여부 표시
- 퀘스트 패널에서 완료/진행/대기 상태를 더 명확히 구분

우선도: 중상. 이미 퀘스트 콘텐츠가 많으므로 안내 UX의 효과가 크다.

## 로그라이크 기본기로 추가하면 좋은 항목

### 성장 루프

현재 성장은 장비, 소모품, 퀘스트 보상 중심이다. 장기 플레이 동기를 강화하려면 경험치 또는 레벨업이 필요하다.

후보 설계:

- 몬스터 처치 시 XP 획득
- 레벨업 시 HP/MP 회복 및 최대치 증가
- 공격/방어/속도 중 하나 선택
- 퀘스트 완료 보상으로 XP 지급

우선도: 중간. 게임의 반복 동기를 강화하지만, 밸런싱 비용이 따른다.

### 원거리 전투와 타겟팅

현재 활은 장비 종류로 존재하지만 전투는 근접 이동 충돌 중심이다. 원거리/마법이 들어가면 전투 선택지가 확장된다.

후보 설계:

- `f`로 타겟팅 모드 진입
- 가장 가까운 보이는 몬스터 자동 선택
- 방향키/Tab으로 대상 변경
- Enter로 발사, Esc로 취소
- 활 장착 시 사거리와 피해 보정 적용
- MP를 사용하는 주문으로 확장 가능

우선도: 중간. 전투 깊이를 크게 늘리지만 UI/AI/밸런스와 같이 설계해야 한다.

### 아이템 조작 확장

현재 아이템은 자동 픽업과 장비/소모품 사용 중심이다. 로그라이크 기본 상호작용을 늘리려면 아이템 관리 기능이 더 필요하다.

후보 기능:

- 바닥 아이템 목록
- 선택 픽업
- 아이템 버리기/drop
- 소모품 퀵슬롯
- 수량 선택
- 장비 비교 표시

우선도: 중간. 인벤토리 복잡도가 커지기 전 UX 틀을 잡는 것이 좋다.

### 휴식과 회복

`Space` 대기는 있지만, 여러 턴 휴식하거나 안전할 때까지 회복하는 흐름은 없다.

후보 기능:

- `.` 또는 `Space`: 한 턴 대기 유지
- `R`: 안전할 때까지 휴식
- 적이 시야에 들어오거나 HP가 가득 차면 자동 중단
- 휴식 중 HP/MP 자연 회복 규칙

우선도: 중간. 던전 탐험 템포와 자원 관리가 생긴다.

## 깊이를 더하는 후순위 항목

### 지형과 상호작용

현재 타일 의미는 주로 `Wall`/`Floor`에 머문다. 맵 생성기는 다양한데, 타일 상호작용은 아직 단순하다.

후보:

- 문, 잠긴 문, 열쇠
- 함정
- 물, 독 늪, 불, 얼음
- 시야를 가리는 안개/연기
- 파괴 가능한 벽 또는 장애물

### 상태이상

전투 결과가 HP 감소 중심이라 장기적인 전술 변수가 적다.

후보:

- 독, 출혈, 화상
- 둔화, 가속
- 기절, 침묵
- 방어력 감소, 취약
- 축복/저주 아이템

### 로그/메시지 히스토리

하단 로그는 최근 몇 줄만 보여준다. 전투와 퀘스트 메시지가 늘어나면 과거 메시지를 다시 확인할 수 있어야 한다.

후보:

- 로그 히스토리 패널
- 중요도별 색상
- 반복 메시지 압축
- 퀘스트/전투/시스템 필터

### 설정과 접근성

이미 글리프 스타일 전환이 있어 좋은 출발점이다. 장시간 플레이를 위해 설정 범위를 넓힐 수 있다.

후보:

- 폰트 크기 조절
- 애니메이션 속도 조절
- 색각 보정 팔레트
- 키 리맵
- 로그 표시 줄 수 조절

## 추천 구현 순서

1. Game Over 오버레이와 새 게임 흐름
2. 인게임 도움말 패널
3. 상단 HUD / 상태 요약
4. 퀘스트 목표 안내 개선
5. 원거리 타겟팅 또는 레벨업 중 하나 선택
6. 아이템 조작 확장
7. 휴식/회복 규칙
8. 지형 상호작용과 상태이상

이 순서는 새 콘텐츠를 많이 추가하기보다, 이미 구현된 시스템을 플레이어가 더 잘 이해하고 반복 플레이할 수 있게 만드는 방향을 우선한다.
